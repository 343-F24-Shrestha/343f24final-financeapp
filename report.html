Project Requirements Report

How Our Project Meets the Requirements

  1. This is the URL to our GitHub repository, shared with each other and the professor: GitHub Repository
  2. This is the URL to our deployed application where anyone can try our app: Deployed Application
  3. Our site uses media queries to support distinct layouts for large and small screens. You can see this in our CSS files, where media queries control images and switch between rows and columns.
  4. We have at least 6 separate pages, each with a meaningful purpose:
    • Index Page (Home) click the buttons to go to annual income Page
    • Annual Income Page, click the buttons to go to money left page
    • Money left Page, click the buttons to go to low risk page
    • Low Risk Investing page, Click the Click here for high risk page
    • High-Risk Investment, click at the bottom in the footer the click here to view reconsider investing page
    • Reconsider Investing, this is the last page, you can click the button in the footer to go to the home page
  5. Each page includes a header to introduce the page content. Headers are styled in each CSS file with background colors and text styles.
  6. Our site includes navigation links that allow users to move between pages in a meaningful flow. For example, on each page, there are links back to the homepage and to related pages.
  7. We include images with appropriate alt attributes for accessibility and to aid users with images disabled or using screen readers, just look here.
  8. We validated each page and stylesheet using the Nu Validator and the WAVE Web Accessibility Evaluator. No warnings or errors were reported.
  9. Our site uses at least two different font faces. We use a sans-serif font for headings and a serif font for paragraph text to create visual distinction.
  10. We included a form on our main page (index.html). This form prompts the user to select their age group. The form includes basic content verification for demonstration purposes.
  11. We included external links on multiple pages. For example, there will be links on the high risk and low risk pages and other pages within our report demonstrate this requirement.
  12. For the next milestone's interactivity, we are planning to use the following APIs:
    • Giphy API for fetching GIFs based on investment topics.
    • Spotify Web API to suggest investment-related podcasts or motivational content.
